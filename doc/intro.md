# 一、产品主要页面（Web + Mobile 通用）

你最终版本会包含以下主视图：

1. 今日任务 Today
2. 本周任务 This Week
3. 全部任务 All Tasks
4. 日历视图 Calendar

   * 月视图 Month View
   * 周视图 Week View

---

# 二、Web 端（桌面端）原型草图

Web 端采用三栏结构（左侧导航、中间列表、右侧详情）。

## 2.1 顶级布局（Web）

```
+--------------------------------------------------------------------------------+
| Header（可选）  LOGO / 折叠菜单按钮 / 设置                                      |
+------------------------+--------------------------------+----------------------+
| 左侧导航栏             |   中间任务列表区域             | 右侧任务详情（可折叠） |
| （固定宽度）           |   （主操作区）                 |                      |
|------------------------+--------------------------------+----------------------|
| [今日 Today]           |  [页面标题]                    |  Task Title          |
| [本周 This Week]       |  -----------------              |  ------------------- |
| [全部 All]             |  [+ 新建任务输入框]            |  描述 description    |
| [日历 Calendar]        |  -----------------              |  日期 date           |
|                        |  [任务列表 List]               |  时间 start-end      |
|                        |    - 09:00 会议                |  状态 pending/done   |
|                        |    - 全天 买菜                 |  [保存] [删除]       |
|                        |    - 14:00 写方案              |                      |
+------------------------+--------------------------------+----------------------+
```

右侧「详情栏」在宽屏下显示，窄屏下会改为 Drawer 弹出。

---

## 2.2 Web — 月视图（Month View）草图

```
[<]  2025年1月  [今天]  [>]                [ 月视图 | 周视图 ]

   日    一    二    三    四    五    六
+-----+-----+-----+-----+-----+-----+-----+
| 30  | 31  |  1  |  2  |  3  |  4  |  5  |
|     |     | ◎2 |     | ○1 |     |     |
|     |     |任务1...   |任务 |     |     |
+-----+-----+-----+-----+-----+-----+-----+
|  6  |  7  |  8  |  9  | 10  | 11  | 12  |
| ○1 | ◎3 | ○1 |     |  +2 | ... |     |
+-----+-----+-----+-----+-----+-----+-----+
| ...（后续行）                                                |
+----------------------------------------------------------------+
下方区域（可选显示当日任务列表或右侧详情）
```

符号说明：

* `◎`：显示任务条，如「09:00 会议」
* `○`：简化显示，任务数量
* 超过可显示区域：`+3`

操作交互：

* 点击格子 → 中间列表切换到该日任务
* 双击空白 → 弹出新建任务（日期自动填入）

# 三、移动端原型草图（Mobile）

移动端采用单列布局 + 底部导航，任务详情用全屏页面或 Drawer。

---

```

| AppBar：< 返回 | 今日任务 |  [日历]                |
+-----------------------------------------------------+

| [x] 14:00 写方案                                    |

底部导航栏

| 今日    | 本周     | 全部     | 日历      |
+---------+----------+----------+-----------+
```

## 3.2 移动端 — 月视图草图

```
+----------------------------------------------+
| < 2025年1月 >         [ 月 | 周 ]            |
+----------------------------------------------+

日 一 二 三 四 五 六
-----------------------
|30|31| 1| 2| 3| 4| 5|

|○1|◎3|○1|  | +2|   |

下方显示选中日期任务：
+----------------------------------------------+
| 1月10日（3个任务）                            |
```

交互总结：

* 点击日期格子 → 下方列表刷新对应日期任务
* 点击任务 → 打开任务详情页
* 点击右下角 `+` → 创建任务（自动填入日期）

# 四、总结：完整交互规范（一句话概括）

## 4.1 列表视图

* Today / Week / All 统一在中间区域显示列表
* 点击任务 → 打开详情（Web: 右侧栏 | Mobile: 全屏）

## 4.2 日历月视图

* 月格子中展示任务数量或前几条任务
* 点击日期 → 列出当天任务
* 双击空白格 → 新建任务

## 4.3 日历周视图

* 时间段任务以「块」显示在时间表中
* 全天任务显示在顶部区域
* 点击空白 → 新建任务（自动带入时间段）

## 4.4 任务详情

* Web：右侧固定栏 / Drawer
* Mobile：全屏页面
* 可编辑：标题、描述、日期、时间段、状态
