# 主任务进度与子任务展示交互说明（补充文档）


## 2. 核心设计原则

* 主任务表达“要完成什么”
* 子任务表达“如何完成”
* 主任务在列表中可被快速扫描
* 子任务仅在主任务上下文中出现
* 通过进度建立主任务与子任务的强关联

---

## 3. 主任务的展示内容

### 3.1 列表页中的主任务展示

在任务列表中，主任务以“结果导向”的信息为主，仅展示对决策有价值的关键信息。

#### 展示内容

* 完成勾选框
* 主任务标题
* 任务进度（仅当存在子任务时显示）
* 日期信息（今天 / 明天 / 具体日期 / 已过期等）

#### 展示结构示例

```
[完成勾选]  主任务标题    2 / 5    今天
```

#### 展示规则说明

* 列表页不展示主任务的描述内容
* 列表页不展开显示子任务列表
* 主任务始终以单行形式展示，避免信息密度过高

---

### 3.2 任务详情页中的主任务展示

在右侧任务详情面板中，主任务展示完整信息，用于查看与编辑。

#### 展示内容

* 标题
* 进度（如存在子任务）
* 描述
* 日期 / 周期
* 四象限（重要性与紧急性）

主任务在详情页中承担“容器”角色，用于承载和管理子任务。

---

## 4. 子任务的展示内容

### 4.1 子任务的展示位置

子任务仅在**主任务详情页**中展示，作为主任务的一部分存在，不在以下位置单独出现：

* 今日列表
* 未来列表
* 已完成列表
* 任何独立任务视图

该设计用于强化用户认知：**子任务是步骤，而不是独立任务**。

---

### 4.2 子任务的展示形式

子任务以简化列表形式展示，每一项仅包含执行所需的最小信息。

#### 展示内容

* 子任务完成勾选框
* 子任务标题

#### 展示示例

```
☐ 准备材料
☑ 联系相关人员
☐ 完成初稿
```

#### 展示规则说明

* 子任务不展示日期、优先级、四象限等属性
* 子任务不显示独立状态标签
* 子任务不在系统中作为独立待办项被统计

---

### 4.3 子任务与主任务的视觉区分

为避免子任务与主任务混淆，子任务在视觉上需满足以下要求：

* 相较主任务标题，字号略小
* 整体向右缩进，体现从属关系
* 仅使用基础完成态样式，不使用强调色

子任务完成后，仅影响主任务进度，不影响任务列表中其他任务的展示与排序。

---

## 5. 主任务与子任务的角色区分总结

| 维度       | 主任务     | 子任务  |
| -------- | ------- | ---- |
| 表达含义     | 目标 / 结果 | 执行步骤 |
| 列表页展示    | 是       | 否    |
| 详情页展示    | 是       | 是    |
| 是否可独立存在  | 是       | 否    |
| 是否参与列表分组 | 是       | 否    |
